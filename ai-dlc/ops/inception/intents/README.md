# Intents

An **Intent** is a high-level description of a capability or outcome the team wants the system to have. It is the starting point for every feature.

## What an Intent Is Not

- Not a user story — it does not have to fit a "as a X I want Y so that Z" structure
- Not a task — it does not describe implementation steps
- Not a specification — the specification emerges from mob elaboration

## How to Create an Intent

1. Copy `_template.md` to a new file: `YYYY-MM-DD-intent-name.md`
2. Fill in the What, Why, and Success sections
3. Leave Elaboration Sessions blank — these are filled in during mob elaboration
4. Set Status to `Draft`

## Intent Status Lifecycle

| Status | Meaning |
|---|---|
| `Draft` | Being written, not ready for elaboration |
| `Ready` | Complete enough to run a mob elaboration session |
| `Elaborated` | All units extracted, added to the backlog |
| `Deferred` | Postponed — recorded for future reference |
