## Discussion Notes

### Key Decisions

| Decision | Reasoning |
|---|---|
| All derived CSS variables harmonised in Unit 1 (not a separate clean-up unit) | Avoids partial theming visible between units; simpler to review in one diff |
| Image compressed under 500 KB before committing | Prevents slow login page load; free-license image from Unsplash |
| `AuthenticatedLayout` wraps all protected routes in `App.tsx` | Cleanest structural approach; keeps auth redirect logic in `ProtectedRoute` |
| Header/footer text uses dark orange (not white/cream) | Engineer preference; WCAG AA contrast to be verified against chosen shade |
| Orange anchor color to be proposed during unit execution | No existing brand color; palette will be defined before implementation starts |
| Dark mode out of scope | Summer theme is light mode only |
| "Cabin Connect" (with space) as the display name in the header | Canonical brand name with space |

### Deferred / Out of Scope

|---|---|

- `AuthenticatedLayout` must compose with `ProtectedRoute`, not replace it
