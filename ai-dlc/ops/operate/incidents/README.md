# Incidents

Production issues, bugs caught post-merge, or unexpected behaviour in the live system. File naming: `YYYY-MM-DD-<short-slug>.md`.

## Severity Levels

| Level | Meaning |
|---|---|
| `P1` | System down or data loss — respond immediately |
| `P2` | Core feature broken for users — resolve within hours |
| `P3` | Degraded experience — resolve within a bolt |
